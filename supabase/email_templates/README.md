# Налаштування листів підтвердження email — покрокова інструкція

## Чому це потрібно зробити, а не просто "красиво"

Зараз Qorax використовує вбудований email-сервіс Supabase. У нього два
обмеження, які напряму впливають на продакшен:

1. **Лімит 2 листи на годину.** Якщо за годину зареєструється 3+ людей —
   третій і далі просто не отримають лист підтвердження, без помилки на
   екрані (тиха відмова). Це функціональний блокер, не косметика.
2. **Лист виглядає як технічний Supabase-шаблон**, не як Qorax.

Підключення Custom SMTP (Resend) вирішує обидві проблеми одночасно:
лімит підскакує до 30 нових користувачів/годину, і лист можна повністю
переробити під свій бренд.

## Крок 1 — Реєстрація в Resend (5 хвилин, без домену)

1. Йдеш на resend.com → Sign up (можна через GitHub).
2. Безкоштовний тариф: 3000 листів/місяць, 100/день — більш ніж достатньо
   для старту.
3. У Dashboard → API Keys → Create API Key. Збережи ключ, він показується
   один раз.

## Крок 2 — SMTP-креденшали Resend

Resend дає SMTP-доступ "з коробки", без додаткових налаштувань:

- Host: `smtp.resend.com`
- Port: `465` (SSL) або `587` (TLS)
- Username: `resend` (буквально це слово, не email)
- Password: API-ключ з кроку 1

## Крок 3 — Custom SMTP у Supabase

Supabase Dashboard → твій проєкт → Authentication → Settings →
SMTP Settings (іноді розділ називається "Email" залежно від версії UI):

1. Увімкнути "Enable Custom SMTP".
2. Вставити host/port/username/password з кроку 2.
3. **Sender email**: `onboarding@resend.dev` (тимчасово, без домену —
   див. примітку нижче).
4. **Sender name**: `Qorax`.
5. Save.

Після збереження новий лімит auth-листів — 30/годину. Якщо потрібно
більше (наприклад, запуск з рекламою), Authentication → Rate Limits →
підняти значення вручну.

## Крок 4 — Кастомний шаблон листа

Supabase Dashboard → Authentication → Email Templates → **Confirm signup**:

1. Subject heading: `Підтвердьте email для Qorax`
2. Message body (HTML): вставити повністю вміст файлу
   `supabase/email_templates/confirm_signup.html` з цього репозиторію.
3. Save.

Шаблон вже відповідає дизайн-системі Qorax (темний фон, лайм-кнопка,
лого з бар-смужками) і використовує плейсхолдер `{{ .ConfirmationURL }}`,
який Supabase сам підставляє з правильним токеном — нічого додатково
кодити не треба.

## Важлива примітка про onboarding@resend.dev

Це адреса Resend для тестування, **не для постійного продакшену**.
Resend прямо попереджає, що листи з неверифікованого домену частіше
потрапляють у спам і виглядають менш надійно для отримувача. Поки немає
свого домену — це найкращий доступний варіант (і він уже значно краще
за дефолтний Supabase-сервіс), але правильне рішення — придбати домен
(~$10-12/рік) і верифікувати його в Resend (Resend Dashboard → Domains →
Add Domain → додати DNS-записи SPF/DKIM, які видасть Resend).

**Рекомендація:** з огляду на функціональний лімітер (2 листи/годину) і
ризик спаму без домену, варто розглянути покупку домену раніше, ніж
"після першого продажу", як було заплановано — ціна (~$10-12/рік)
мінімальна порівняно з ризиком втратити реальних перших клієнтів через
лист, що не дійшов або потрапив у спам.

## Після підключення домену (коли буде)

Жодних змін у коді не потрібно:
1. Resend Dashboard → Domains → Add Domain → додати DNS-записи.
2. Supabase SMTP Settings → змінити Sender email на `noreply@qorax.com`
   (чи інша адреса на твоєму домені).

Це все — лист одразу стане повністю "своїм" і з кращою доставлюваністю.
